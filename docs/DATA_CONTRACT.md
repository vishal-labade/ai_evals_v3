# DATA CONTRACT --- AI Evals v2

This document defines the expected schema and invariants for AI Evals
v2.

The system intentionally separates:

1.  Behavioral reliability evaluation (v2_suite_50.jsonl)
2.  Context-boundary memory evaluation (v2_ctx_cliff_suite_20.jsonl)

Both tracks share the same execution and logging infrastructure, but
produce different metric outputs.

------------------------------------------------------------------------

# 1. Scenario JSONL Schema

Each line in a scenario file (e.g., v2_suite_50.jsonl,
v2_ctx_cliff_suite_20.jsonl) must contain:

Required fields:

-   `scenario_id` (string)
-   `tags` (array\[string\])
-   `checks` (object)
-   `turns` (array)

The `checks` object may contain:

-   `persona`
-   `memory`
-   `format`
-   `recovery`
-   `calibration`
-   `consistency`

Only checks defined for a scenario are scored.

------------------------------------------------------------------------

# 2. Run JSONL Schema

Each model execution produces a run JSONL file in:

-   `results/v2/runs/behavioral/`
-   `results/v2/runs/ctx_cliff/`

Each record (per turn) must contain:

-   `run_id` (string)
-   `scenario_id` (string)
-   `turn_index` (int)
-   `assistant_text` (string)
-   `model` (string)
-   `profile` (string)
-   `options` (object: may include `temperature`, `num_ctx`,
    `num_predict`)

Invariant:

-   A single `run_id` corresponds to exactly one model + profile +
    configuration.
-   `scenario_id` must exist in the scenario file used for scoring.

------------------------------------------------------------------------

# 3. Behavioral Metrics Output

Generated by:

-   `scripts/v2/scorecard.py`

Primary output:

`behavioral_metrics.csv`

Columns:

-   `run_id`
-   `profile`
-   `model`
-   `temperature`
-   `num_ctx`
-   `scenario_id`
-   `tags`
-   `persona_pss`
-   `persona_drift_turn`
-   `format_score`
-   `memory_score`
-   `recovery_score`
-   `contradiction_rate`
-   `avg_confidence`
-   `avg_abs_error`
-   `bri`

Definitions:

-   `persona_pss` --- fraction of persona-required turns satisfied
-   `persona_drift_turn` --- first persona violation turn
-   `memory_score` --- short-horizon recall accuracy
-   `recovery_score` --- correction success rate
-   `contradiction_rate` --- contradiction frequency
-   `bri` --- mean of available behavioral components for that scenario

Aggregated scorecards:

-   `model_scorecard.json`
-   `profile_scorecard.json`
-   `model_profile_scorecard.json`

------------------------------------------------------------------------

# 4. Context Cliff Metrics (Memory Compliance)

Generated by:

-   `memory_compliance.py`

Run-level output:

`metrics_run_mcs.csv`

Columns:

-   `run_id`
-   `mcs`
-   `drop_turn_mean`
-   `mcs_auc`
-   `mcs_n`

Definitions:

-   `mcs` --- fraction of required memory checks satisfied
-   `drop_turn_mean` --- first required turn index where memory fails
-   `mcs_auc` --- area under MCS-by-turn curve
-   `mcs_n` --- number of evaluated checkpoints

Turn-level output:

`metrics_turn_mcs.csv`

Columns:

-   `run_id`
-   `scenario_id`
-   `turn_index`
-   `mcs_turn`

------------------------------------------------------------------------

# 5. Enriched Metrics Output

Generated by:

-   `enrich_metrics_with_run_metadata.py`

Enrichment appends metadata:

-   `profile`
-   `model`
-   `cell_id`
-   `family`
-   `temperature`
-   `num_ctx`
-   `num_predict`
-   `host`
-   `run_file`

Enrichment must not modify metric values.

------------------------------------------------------------------------

# 6. Frozen Benchmark Policy

-   `v2_suite_50.jsonl` is a frozen behavioral benchmark.
-   `v2_ctx_cliff_suite_20.jsonl` is a frozen context-boundary
    benchmark.

Future revisions require a new versioned suite and fresh metrics.

------------------------------------------------------------------------

# 7. Reproducibility Guarantees

A run is valid if:

-   The frozen scenario file is used.
-   Model configuration is logged.
-   `num_ctx`, `temperature`, and `num_predict` are recorded.
-   Metrics are computed deterministically.

No metric depends on manual annotation.

End of data contract.
